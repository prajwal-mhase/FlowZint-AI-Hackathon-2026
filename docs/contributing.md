# Contributing

## Development Standards

- Keep modules small and feature-oriented.
- Prefer typed contracts at service boundaries.
- Add tests for behavior that can regress.
- Keep secrets out of source control.
- Use migrations for database changes.
- Document architectural decisions when they affect future phases.

## Commit Style

Use concise, imperative commit messages:

```text
feat: add document upload service
fix: validate organization access for conversations
docs: explain RAG architecture
```
